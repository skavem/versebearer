# build

## Purpose
Wails3 build pipeline. Per-OS Taskfiles, Windows NSIS installer setup, Linux nfpm packaging + AppImage builder, macOS Info.plist, app icons, and the master `info.json` consumed by `wails3 generate syso`. Driven by the root `Taskfile.yml` which `includes:` these files.

## Key Files
| File | Description |
|------|-------------|
| `Taskfile.common.yml` | Shared tasks: `go:mod:tidy`, `install:frontend:deps`, `build:frontend`, `generate:bindings`, `generate:icons`, `dev:frontend`, `update:build-assets` |
| `Taskfile.windows.yml` | Windows `build` (CGO=0, `-tags production -trimpath -ldflags="-w -s -H windowsgui"`), `generate:syso`, `create:nsis:installer`, `run` |
| `Taskfile.darwin.yml` | macOS build + `.app` bundle |
| `Taskfile.linux.yml` | Linux build, AppImage, deb/rpm via nfpm |
| `config.yml` | Wails3 build config: app metadata (still default "My Company / My Product" — needs updating), `dev_mode` watcher rules, file associations |
| `info.json` | Windows VERSIONINFO data consumed by `wails3 generate syso` |
| `Info.plist`, `Info.dev.plist` | macOS bundle plists |
| `appicon.png` | Source icon, converted to `.ico`/`.icns` via `wails3 generate icons` |
| `icon.ico`, `icons.icns` | Generated app icons (Windows / macOS) |
| `wails.exe.manifest` | Windows executable manifest |

## For AI Agents

### Working In This Directory
- `config.yml` still has placeholder values (`My Company`, `My Product`, `com.mycompany.myproduct`). If you change those, run `task common:update:build-assets` to regenerate per-OS asset files — that command **overwrites manual edits** to the generated files (Info.plist, nsis, nfpm), so it's a one-way sync from `config.yml`.
- Windows build embeds the manifest + icon via a `.syso` file generated by `wails3 generate syso` (sourced from `icon.ico`, `wails.exe.manifest`, `info.json`). The `.syso` is deleted post-build (`Remove-item *.syso` / `rm -f *.syso`).
- Production build flags: `-tags production -trimpath -ldflags="-w -s -H windowsgui"`. The `windowsgui` linker flag hides the console window on Windows.
- Dev mode (`config.yml` `dev_mode`): watches `*.go` files, runs `wails3 task common:install:frontend:deps` (once), `common:dev:frontend` (background Vite), then `go mod tidy` → `build` → `run` chain.
- Vite dev port is 9245 by default (root `Taskfile.yml` `VITE_PORT`). Override via env: `WAILS_VITE_PORT=NNN task dev`.
- Bindings generation: `task common:generate:bindings` runs `wails3 generate bindings -d ./frontend/src/lib/bindings -f '{{.BUILD_FLAGS}}'`. The `-ts` flag is gated by `UseTypescript` — not set in the current config but the bindings ARE TypeScript files. Investigate before "fixing".
- The `package` task on Windows depends on `build` with `PRODUCTION: "true"` — packaging triggers a fresh production build, not reuse of a debug binary.

## Dependencies

### External (build-time)
- `wails3` CLI (bindings, syso, icons, build-assets).
- `task` (go-task).
- `go` toolchain (1.22+ per `go.mod`).
- `npm` (frontend) and bun-or-equivalent (reciever).
- Per-OS: NSIS (`makensis`), nfpm, AppImage tools, `lipo` (macOS universal).
